## 🤖 AI Project

<details open>
<summary><b>🧠 TestCraft AI — AI-Powered Test Case Generator</b> &nbsp; <img src="https://img.shields.io/badge/Status-Active-brightgreen?style=flat-square"/> <img src="https://img.shields.io/badge/AI-Powered-blueviolet?style=flat-square&logo=openai&logoColor=white"/></summary>
<br/>

> 🚀 **An intelligent test case generation tool built for QA engineers — powered by AI**

**The Problem:** Writing comprehensive test cases manually is time-consuming, repetitive, and prone to human oversight. A single feature can require dozens of edge cases that engineers often miss under deadline pressure.

**The Solution:** TestCraft AI takes a feature description or user story as input and automatically generates structured, ready-to-use test cases — covering happy paths, edge cases, negative scenarios, and boundary values.

---

### ✨ Key Features

| Feature | Description |
|---|---|
| 📝 **Smart Input Parsing** | Accepts user stories, feature specs, or plain descriptions |
| 🔄 **Full Coverage Generation** | Auto-generates positive, negative, boundary & edge cases |
| 📋 **Structured Output** | Exports test cases in Excel / JSON / Markdown format |
| 🧩 **Module-Aware** | Understands modules like Login, Cart, Checkout, API endpoints |
| ⚡ **Instant Results** | Generates 20–50 test cases in seconds |
| 🔐 **Security Test Hints** | Flags fields that need SQL injection / XSS checks |

### 🔁 How It Works

```
User Story / Feature Description
         ↓
   AI Prompt Engine
         ↓
   Test Case Generator
   ┌─────────────────────────────────┐
   │ ✅ Positive / Happy Path Cases  │
   │ ❌ Negative / Failure Cases     │
   │ 🔢 Boundary Value Cases         │
   │ 🔐 Security Edge Cases          │
   └─────────────────────────────────┘
         ↓
   Structured Output (Excel / JSON / MD)
```

---

### 📊 Sample Output

**Input:** *"User should be able to login with email and password"*

| TC ID | Title | Steps | Expected Result | Type |
|---|---|---|---|---|
| TC_001 | Valid login | Enter correct email & password → Click Login | Redirect to dashboard | ✅ Positive |
| TC_002 | Wrong password | Enter valid email, wrong password → Click Login | Show "Invalid credentials" | ❌ Negative |
| TC_003 | Empty email | Leave email blank → Click Login | Show "Email required" | ❌ Negative |
| TC_004 | SQL Injection | Enter `' OR 1=1--` in email field | Login blocked, no data exposed | 🔐 Security |
| TC_005 | Max length email | Enter 256-char email → Click Login | Graceful validation error | 🔢 Boundary |
